# 需求概述

这是一个以搭建的标准商品库为核心，通过对多种媒体数据来源进行 AI 识别、AI 匹配与权重化统计，持续输出可解释时尚趋势洞察的数据系统。

## 一、需求背景与目标

### 1. 背景

希望系统化地洞察「时尚趋势」：

- 不再只依赖人工主观经验
- 能筛选查看，指定某段时间内，哪些商品、哪些设计维度更热门
- 能查看不同信息来源（名人、时尚媒体、零售商）的影响力（即时尚度）

### 2. 核心目标

构建一套“基于标准商品库的趋势识别与统计系统”，实现：

1. 从媒体数据来源中框选出商品图或提取出商品名  
2. 与标准商品库里的图或文字进行匹配  
3. 在权重机制下进行频次与趋势统计  
4. 以数据看板形式，为业务提供可解释的趋势洞察  

---

## 二、关键业务问题

- 最近 xx 天 / 某几个月内，最火的包款 / 上衣款 / 鞋款是哪些
- 某一品类中，哪些设计维度更流行（比如包类，包型这个设计维度的 tote 频次最高）
- 某个名人 / 零售商 / 时尚媒体的时尚度怎样
- 回顾：发现某个名人 / 零售商的时尚度高或低时，能够调整权重，得出更有参考性的流行趋势统计

---

## 三、核心模块

### 1. 标准商品库

作为全系统唯一的“商品真值来源”，用于匹配。需要与业务人员确认标准库全集选取哪一个网站。

#### 数据来源

- 时尚大牌官方——需要与业务确定具体来源。

#### 需要爬取的信息

**视觉信息**
- 官方商品图片

**商品信息**
- 品牌
- 品类（包 / 鞋等）
- 商品名称 / 款号

**维度信息**
- 包型（Tote / Shoulder / Mini …）
- 材质（Leather / Canvas / Nylon …）
- 颜色（Yellow / Blue / Red …）
- 工艺（Logo / 链条 / 金属件等）

---

### 2. 媒体数据来源（待匹配的数据，可配置权重）

#### 媒体类型

- 社交媒体（名人 ins 帖子）
- 时尚媒体（各个时尚媒体网站爬取的图文）
- 零售商（电商 best seller 页）
- 其他（如狗仔网站）

#### 需要爬取的信息

- 来源（具体账号 / 具体媒体名 / 具体网站）
- 发布时间
- 内容（图 / 文 / 视频 / 图文）

**维度信息（如有）**
- 包型（Tote / Shoulder / Mini …）
- 材质（Leather / Canvas / Nylon …）
- 颜色（Yellow / Blue / Red …）
- 工艺（Logo / 链条 / 金属件等）

**时尚度（初版只包含权重这一维度）**
- 针对具体某个媒体数据来源，可设定时尚度（具体账号 / 媒体名 / 网站）
- 业务人员可配置，直接作用于统计结果

---

### 3. AI 识别与匹配

#### AI 主体识别

- AI 从媒体数据来源中识别一个或多个商品主体  
  （比如从名人 ins 帖子里的网红图，框选和提取出包包衣服等商品；或从一段文字中提取出包包衣服等商品的型号名称）
- 支持图片、文本、图文、视频

#### AI 商品匹配

- AI 根据识别出的商品主体，在对应品类的标准库中进行匹配

#### 维度提取

- 匹配后，获取标准商品库里对应的结构化维度词和商品型号
- 根据匹配结果对流行趋势统计产生直接影响  
  （根据品类，对应的维度词计数增加、对应的商品型号计数增加）

---

### 4. 流行趋势统计（数据统计）

- 可指定时间范围进行流行趋势统计
- 按品类，展示每个品类的高频商品 TOP 10
- 按品类，展示每个品类下高频的维度关键词  
  （每个维度的 TOP 3，比如包包品类的颜色维度）
